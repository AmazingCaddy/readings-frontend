# 前端学习 GitHub Pages 项目启动 Prompt

你现在要帮我从零开始做一份“前端开发进阶学习资料”，目标是输出成 GitHub Pages，方便我系统学习和准备前端岗位面试。

## 我的背景

我有比较多开发经验，熟悉前端和客户端开发，也做过一些简单后端。但我希望这份前端资料不是入门教程，而是面向“有开发经验、希望补齐系统性前端知识、准备中高级前端面试”的学习材料。

我希望内容能覆盖前端工程里的核心概念、原理、实战写法、常见坑、系统设计和面试表达。

## 你要怎么工作

请使用 goal 模式来做，每个 goal 只收敛一个阶段，不要一次性把所有内容都做完。

推荐节奏：

1. 先建立 GitHub Pages / Docusaurus 项目并完成基础导航。
2. 然后和我讨论整体学习大纲。
3. 再按模块拆 goal 补文章。
4. 每篇文章完成后运行构建验证。
5. 构建没问题再单独 commit。
6. 不要等整个列表全部写完才 commit。

如果当前目录还没有项目，请初始化一个适合 GitHub Pages 的文档站点。优先使用 Docusaurus，内容使用 Markdown，支持 Mermaid 图表。

## 内容风格要求

每篇文章都要让我真正知道：

- 这个技术是什么。
- 为什么需要它。
- 它解决什么问题。
- 它在真实项目里怎么用。
- 不这么做会导致什么问题。
- 常见坑是什么，怎么规避。
- 面试时怎么表达。

不要只列概念，也不要只写抽象叙述。需要有：

- 图表，比如 Mermaid 流程图、时序图、状态图。
- 代码示例，至少要有伪代码；适合前端的地方要给 TypeScript / JavaScript / React 示例。
- 真实例子，比如一个页面、一个组件、一个请求链路、一个构建问题、一个性能问题。
- 对比反例，说明错误做法会造成什么后果。
- 失败处理或兜底策略，比如请求失败、缓存失效、状态错乱、构建失败、兼容性问题。
- 引用链接，优先引用官方文档和高质量资料。

文章读者是我这种有开发经验但希望系统补齐前端深度的人，所以不要写成零基础教程，也不要跳过关键术语。遇到术语要解释清楚，让我看到词就知道大概是什么意思、怎么实现、为什么重要。

## 建议章节方向

你可以先基于下面方向设计大纲，然后和我确认：

### 1. 浏览器与 Web 基础

- 浏览器渲染流程：HTML/CSS/JS 如何变成页面。
- 事件循环、宏任务、微任务、渲染时机。
- HTTP 缓存、强缓存、协商缓存、Service Worker 缓存。
- 跨域、CORS、Cookie、SameSite、Storage。
- Web 安全：XSS、CSRF、点击劫持、CSP。

### 2. JavaScript 与 TypeScript 深入

- 原型链、闭包、作用域、this。
- Promise、async/await、异常传播。
- 模块系统：ESM、CommonJS、Tree Shaking。
- TypeScript 类型系统、泛型、类型收窄、工程实践。
- 内存泄漏、垃圾回收、性能陷阱。

### 3. React 与现代 UI 架构

- React 渲染模型、Fiber、并发渲染基本概念。
- Hooks 原理、依赖数组、闭包陷阱。
- 状态管理：本地状态、Context、Redux/Zustand、Server State。
- 表单、列表、虚拟滚动、复杂交互组件。
- 组件设计：受控/非受控、组合、可访问性。

### 4. 前端工程化

- Vite/Webpack/Rspack 的构建流程。
- Babel、SWC、PostCSS、CSS Modules、Tailwind。
- Monorepo、包管理器、依赖版本治理。
- ESLint、Prettier、TypeScript、测试与 CI。
- Source Map、环境变量、构建产物分析。

### 5. 性能优化

- Core Web Vitals：LCP、INP、CLS。
- 首屏优化、代码分割、预加载、懒加载。
- 图片优化、字体优化、资源优先级。
- 长任务拆分、Web Worker、虚拟列表。
- 性能监控、埋点、线上排障案例。

### 6. 前端可靠性与数据流

- 请求状态管理、重试、超时、取消、竞态控制。
- 乐观更新、回滚、幂等提交、防重复点击。
- 前端缓存与后端缓存如何配合。
- 实时通信：WebSocket、SSE、轮询。
- 离线、弱网、错误边界、降级体验。

### 7. 前端架构与系统设计

- BFF 与前端协作。
- 微前端适用场景、隔离、路由、状态共享。
- 权限系统、菜单系统、动态路由。
- 国际化、主题系统、设计系统。
- 大型表格、低代码页面、可视化编辑器、IM 前端、直播间前端等案例。

### 8. 面试与项目实践

- 前端面试 30 天路径。
- 高频八股题背后的真实工程问题。
- 前端系统设计题：后台管理系统、IM、低代码、埋点 SDK、组件库、性能监控平台。
- 项目复盘模板：背景、难点、方案、权衡、指标、结果。

## 文章质量标准

每篇文章尽量包含这些固定小节：

```text
# 标题

## 场景
说明真实业务问题。

## 是什么
解释概念和核心术语。

## 为什么需要
说明没有它会遇到什么问题。

## 推荐做法
给流程图、时序图或状态图。

## 代码示例
给 TypeScript / JavaScript / React / 伪代码。

## 反例与后果
说明常见错误做法和实际后果。

## 常见坑
列出实战中容易踩的问题。

## 排查与验证
说明如何 debug、如何确认方案生效。

## 面试怎么讲
给一段可以直接用于面试表达的话。

## 延伸阅读
放官方文档或高质量引用链接。
```

## 提交和验证要求

请严格执行：

- 每篇文章完成后运行 GitHub Pages 构建，例如 `npm run build`。
- 构建通过后再 commit。
- 每篇或每个小块单独 commit。
- commit message 使用清晰英文，例如 `docs: add browser rendering guide`。
- 如果构建失败，先修复再 commit。
- 不要提交构建失败的内容。
- 不要把所有文章一次性写完再提交。

## 我希望你先做什么

请先启动一个 goal，目标只收敛到：初始化前端学习资料 GitHub Pages 项目，完成基础文档站点、导航结构、首页/简介页、构建验证和首个 commit。

完成这个 goal 后，先停下来总结当前状态，再和我讨论下一阶段的文章 plan。
