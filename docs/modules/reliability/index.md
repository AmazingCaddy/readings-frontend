# 前端可靠性与数据流

本模块关注真实业务中最容易出问题的数据链路：请求失败、并发竞态、弱网、重复提交、缓存失效和状态回滚。

## 计划文章

- 请求状态管理：loading、empty、error、success。
- 重试、超时、取消请求和竞态控制。
- 乐观更新、回滚、幂等提交和防重复点击。
- 前端缓存与后端缓存协作。
- WebSocket、SSE、轮询和实时数据一致性。
- 离线、弱网、错误边界和降级体验。

## 面试目标

能把“接口请求”讲成完整系统：如何避免脏数据、错序响应、重复提交、不可恢复错误和用户体验断层。
